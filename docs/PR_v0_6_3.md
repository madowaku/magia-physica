# v0.6.3 CardEffects抽出

## 変更内容
- `scripts/battle/CardEffects.gd` を追加し、カード効果処理を `Main.gd` から分離しました。
- 以下のカード効果を `CardEffects` に移動しました。
  - `_apply_push`
  - `_apply_momentum`
  - `_apply_heat`
  - `_apply_recovery`
  - `_apply_scan`
  - `_apply_slip`
  - `_apply_pebble`
  - `_push_amount`
  - `_momentum_damage`
  - `_heat_damage`
  - `_weight_multiplier`
- カード効果による `battle_state` 更新、バトルログ追加、`pending_fx` 用 Dictionary 生成を `CardEffects` 側にまとめました。
- `Main.gd` は「カードを使う」「コストを支払う」「手札から除く」「SFXを鳴らす」「返ってきた `pending_fx` を保持する」役割に寄せました。

## 確認項目
- Godot 4.6.1 で起動し、実行ログにエラーがないことを確認しました。
- ヘッドレス smoke script で以下を確認しました。
  - 「はじめる」ボタンが会話画面へ遷移する
  - 「すぐバトル」ボタンがバトル画面へ遷移する
  - カードの「選択」が動く
  - 全カード種の「発動」が動く
  - 「ターン終了」が動く
  - 勝利後に報酬画面が表示される

## 未対応項目
- UI生成は引き続き `Main.gd` / `UiFactory` 側に残しています。
- Tween演出、`pending_fx` 表示処理、SFX再生は `Main.gd` 側に残しています。
- 敵AI、報酬画面UI、カード報酬内容の大幅変更は今回の対象外です。
- `FxController` / SFX 整理は v0.6.4 で対応予定です。
- 未追跡の patch README、apply scripts、`.import`、`patch_assets/` は今回の変更に含めません。
